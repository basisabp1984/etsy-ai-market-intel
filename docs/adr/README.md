# Architecture Decision Records

This folder records design decisions that were *not obvious* and that a future
maintainer is likely to ask about. We use a light variant of the Michael Nygard
ADR format: context, decision, consequences, status, date.

The format is intentionally lightweight — these are not RFCs. Each ADR explains
one decision in under a page.
